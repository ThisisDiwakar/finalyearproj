## Common Questions

### Q: Which file should I read first?
**A**: Start with `README_PINATA.md` for overview, then `PINATA_QUICK_START.md` for setup.

### Q: I'm stuck, where do I look?
**A**: Check `PINATA_VISUAL_GUIDE.md` for common mistakes, then `PINATA_SETUP_GUIDE.md` troubleshooting section.

### Q: How do I verify it's working?
**A**: Run `node backend/test-ipfs-sync.js` and check `SETUP_COMPLETE.md`.

### Q: Do I need all these files?
**A**: No! Pick one setup guide that matches your style. Others are for reference.

### Q: Can I skip Pinata setup?
**A**: Yes! System works with database fallback. Pinata is optional but recommended for production.

### Q: Where are my API keys stored?
**A**: In `backend/.env` file (never committed to git).

### Q: How do I know if Pinata is connected?
**A**: Backend startup shows "Pinata Connected" instead of "Local Fallback".

## Support

### If you need help:
1. Check the troubleshooting section in any guide
2. Run test script: `node backend/test-ipfs-sync.js`
3. Check backend console logs
4. Review `PINATA_VISUAL_GUIDE.md` for common mistakes
5. Visit Pinata docs: https://docs.pinata.cloud/

### If test script fails:
1. Verify MongoDB is running
2. Check API keys in `.env`
3. Restart backend server
4. Check internet connection

### If backend shows "Local Fallback":
1. Check `.env` has keys
2. Verify no spaces or quotes
3. Restart server
4. Run test script
